## Architecture & Conventions

### Design Philosophy

This project follows a **clean, minimal, modern** design language inspired by Vercel, Linear, and Raycast:

- **Dark-first** — Design for dark mode by default, with thoughtful light mode support.
- **Spatial hierarchy** — Use spacing, typography weight, and subtle borders instead of heavy color blocks.
- **Micro-interactions** — Every interactive element should have a smooth transition (150–300ms ease).
- **Typography** — Use Inter or system fonts. Headings are semibold, body is regular, captions are muted.
- **Color** — Use a restrained palette. Primary accent for CTAs only. Gray scale for structure.
- **Iconography** — Use Lucide icons (web) or SF Symbols–style icons (mobile). Keep them 18–20px.

### Code Style

- **TypeScript** — Strict mode. Prefer `type` over `interface` unless extending.
- **Functional** — Use function components and hooks. No class components.
- **Colocation** — Keep styles, types, and tests next to their component.
- **Barrel exports** — Use `index.ts` files to re-export from feature directories.
- **Naming** — PascalCase for components, camelCase for utilities, SCREAMING_SNAKE for constants.
- **Immutability** — Prefer `const`, spread operators, and `.map()` over mutation.

### File Structure
